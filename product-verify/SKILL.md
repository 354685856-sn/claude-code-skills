# Product Verify Skill

## 用途
验证代码更改是否符合产品需求、设计规范 and 用户体验标准。

## 触发条件
- PR/MR 代码审查时
- 功能实现完成后
- UI/UX 变更时

## 检查清单

### 1. 功能验证
- [ ] 功能是否按需求文档实现
- [ ] 边界条件是否处理
- [ ] 错误场景是否有适当反馈

### 2. 用户体验验证
- [ ] 交互流程是否流畅
- [ ] 错误提示是否友好
- [ ] 加载状态是否有提示

### 3. 设计规范验证
- [ ] 是否符合设计系统
- [ ] 颜色/字体/间距是否一致
- [ ] 响应式布局是否正确

### 4. 可访问性验证
- [ ] 键盘导航是否支持
- [ ] 屏幕阅读器是否兼容
- [ ] 颜色对比度是否达标

## 使用方法

```bash
/claude "使用 product-verify skill 审查这个 PR"
```

## 输出格式

```markdown
## 产品验证报告

### ✅ 通过项
- 列表...

### ⚠️ 需要改进
- 列表...

### ❌ 阻断问题
- 列表...

### 总体评估
[通过/需要修改/重新设计]
```
